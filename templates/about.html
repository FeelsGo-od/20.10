Main Pros of Store made with only Javascript

  • Site is definitely faster than with for example React, because there is no large packages/plugins which can slow down the whole project.
  • More controle of functions - more personalized features
  • No need to update site, because Javascript works great with every version, and versions can be combined

Problems I faced during creation of that project

  • A lot of time gone on mobile version *Tailwind is faster in case of responsive styling
  • Routes are messy with Express.js. *Next.js will be a faster and better looking solution
  • Pug is not a fast way to create dynamic pages, because it has many errors, css does not work correctly *I think rendering components can be a solution to keep less code
  • Sass is actually okay for that moment if it goes about just styling elements
  • Modular architecture really makes error handling process easier. (easier to find bugs)
  • Even more time was wasted on products json data.. *Next time I'll just fetch some from other store
  • My modular code is messy, and some of the media queries in sass as well. *Next.js can be more clear if write it right, I think
  • Sometimes styles names repeating. *In next.js there is no problem with that.
  • But in Next.js can be problems with error debugging, I need to look for some easiest way to do it if it exists.

Why I chose to use Vanilla js for Frontend on that project?

  • Cause I wanted to actually face those problems above and understand roots of web developing, so other frameworks could be more easy to me. So, in the future I will use only required technologies in certain cases.
